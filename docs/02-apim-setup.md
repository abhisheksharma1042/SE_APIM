# 02 — Azure API Management Setup & Identity Linking

---

## Step 1: Provision APIM

### Via Azure Portal

1. Search **API Management** → **Create**
2. Fill in:
   - **Subscription / Resource Group**: your workforce subscription
   - **Resource name**: `contoso-apim` (globally unique)
   - **Region**: same as your backends
   - **Organization name**: display name for the developer portal
   - **Administrator email**: notified of system events
   - **Pricing tier**:
     - `Developer` — for dev/test (no SLA, cheapest)
     - `Basic` — entry-level production
     - `Standard` — includes developer portal, multi-region replication optional
     - `Premium` — multi-region, VNet injection, Availability Zones
3. Click **Review + Create**

> Provisioning takes 30–45 minutes.

### Via Azure CLI (Bicep)

```bash
az group create --name rg-apim-prod --location eastus2

az apim create \
  --name contoso-apim \
  --resource-group rg-apim-prod \
  --publisher-email admin@contoso.com \
  --publisher-name "Contoso" \
  --sku-name Developer \
  --location eastus2
```

Or see the Bicep template in `apim-artifacts/` (generated by APIOps extractor).

## Step 2: Configure the External Identity Provider

This links the APIM Developer Portal login to your Entra External ID tenant.

### Via Portal

1. In your APIM instance → **Developer portal** → **Identities**
2. Click **+ Add**
3. Select **Azure Active Directory B2C** (currently the supported option for external tenants; Entra External ID native support is in preview)
4. Fill in:
   - **Display name**: `External Customers`
   - **Client ID**: `<CLIENT_APP_ID>` (the SPA app you registered)
   - **Client secret**: create a client secret on the app registration
   - **Signin tenant**: `<external-tenant-domain>.onmicrosoft.com`
   - **Authority**: `<external-tenant-domain>.b2clogin.com`
   - **Signin policy**: `B2C_1_susi`
   - **Profile editing policy**: optional
   - **Password reset policy**: optional
5. Save → **Publish** the developer portal

### Via Named Values (for policy use)

Store sensitive values as Named Values in APIM so policies reference them by name:

```bash
# Store external tenant ID
az apim nv create \
  --service-name contoso-apim \
  --resource-group rg-apim-prod \
  --named-value-id "external-tenant-id" \
  --display-name "External Tenant ID" \
  --value "<EXTERNAL_TENANT_ID>" \
  --secret false

# Store API App ID (audience)
az apim nv create \
  --service-name contoso-apim \
  --resource-group rg-apim-prod \
  --named-value-id "api-app-id" \
  --display-name "API App ID (Audience)" \
  --value "<API_APP_ID>" \
  --secret false

# Store client secret as a SECRET named value (backed by Key Vault recommended)
az apim nv create \
  --service-name contoso-apim \
  --resource-group rg-apim-prod \
  --named-value-id "b2c-client-secret" \
  --display-name "B2C Client Secret" \
  --value "<CLIENT_SECRET>" \
  --secret true
```

### Key Vault–Backed Named Values (Recommended for Production)

```bash
# Grant APIM system-assigned identity access to Key Vault
APIM_PRINCIPAL=$(az apim show \
  --name contoso-apim \
  --resource-group rg-apim-prod \
  --query identity.principalId -o tsv)

az keyvault set-policy \
  --name contoso-kv \
  --object-id $APIM_PRINCIPAL \
  --secret-permissions get list

# Reference Key Vault secret in named value
az apim nv create \
  --service-name contoso-apim \
  --resource-group rg-apim-prod \
  --named-value-id "b2c-client-secret" \
  --display-name "B2C Client Secret" \
  --key-vault-id "https://contoso-kv.vault.azure.net/secrets/b2c-client-secret" \
  --secret true
```

## Step 3: Import a Backend API

### From OpenAPI Spec

```bash
az apim api import \
  --service-name contoso-apim \
  --resource-group rg-apim-prod \
  --path "items" \
  --api-id "example-api" \
  --display-name "Example Items API" \
  --specification-format OpenApi \
  --specification-url "https://raw.githubusercontent.com/<org>/<repo>/main/openapi.yaml"
```

### From Azure Function

1. APIM → **APIs** → **+ Add API** → **Function App**
2. Select your Function App → choose functions to import
3. APIM automatically configures the backend URL and creates operations

## Step 4: Create a Product

Products bundle one or more APIs and enforce subscription requirements.

```bash
az apim product create \
  --service-name contoso-apim \
  --resource-group rg-apim-prod \
  --product-id "standard" \
  --product-name "Standard" \
  --description "Standard tier API access" \
  --subscription-required true \
  --approval-required false \
  --state published

# Add the API to the product
az apim product api add \
  --service-name contoso-apim \
  --resource-group rg-apim-prod \
  --product-id "standard" \
  --api-id "example-api"
```

## Step 5: Enable Managed Identity (for backend auth)

Allow APIM to authenticate to backends using its managed identity:

```bash
# Enable system-assigned managed identity
az apim update \
  --name contoso-apim \
  --resource-group rg-apim-prod \
  --enable-managed-identity true
```

Then in APIM policies you can use:
```xml
<authentication-managed-identity resource="https://<backend-resource-id>" />
```

## Step 6: Configure CORS at the Service Level

Add CORS to the global inbound policy (see `policies/global/global-policy.xml`):

```xml
<cors allow-credentials="true">
  <allowed-origins>
    <origin>https://your-spa-domain.com</origin>
    <origin>http://localhost:3000</origin>
  </allowed-origins>
  <allowed-methods preflight-result-max-age="300">
    <method>GET</method>
    <method>POST</method>
    <method>PUT</method>
    <method>DELETE</method>
    <method>OPTIONS</method>
  </allowed-methods>
  <allowed-headers>
    <header>Authorization</header>
    <header>Content-Type</header>
  </allowed-headers>
</cors>
```

## Step 7: Verify the Setup

```bash
# Get the gateway URL
GATEWAY_URL=$(az apim show \
  --name contoso-apim \
  --resource-group rg-apim-prod \
  --query gatewayUrl -o tsv)

echo "Gateway: $GATEWAY_URL"

# Test with a subscription key (unauthenticated — should return 401 after policy applied)
curl -i "$GATEWAY_URL/items"

# Test with a valid B2C token
ACCESS_TOKEN="<token from Step 6 in doc 01>"
curl -i "$GATEWAY_URL/items" \
  -H "Authorization: Bearer $ACCESS_TOKEN"
```
